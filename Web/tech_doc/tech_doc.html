What is the Arduino?

The Arduino commonly refers to the product "Arduino Uno". But is collectively a platform for open source electronic modifications with a suite of microcontroller or microprocessor boards capable of completing many tasks required by the user. The Arduino is also a C/C++ library to interface with boards (library refers to external code that can be used) to program the Arduino board. This means programmers that use Arduino will also need to understand some basic C/C++ syntax.

The Arduino Uno

The guide will use the Arduino IDE (Independent Developing Environemnt) to program Arduino boards and can be downloaded on the Arduino website. In short, the Arduino is a:

  • Hub for all people to learn electronics in a open source environment
  • Collection of electronic boards
  • Software IDE to develop code for the boards
Assumed knowledge

The page will assume that you have never programmed and will explore the most important concepts needed to begin programming in your Arduino. But experienced programmers are always welcomed to skip into relevant sections.

This page will assume that you will have a basic familiarity with:

Variables and data types

Here is a broad overview of some of the data types that exist in the C/C++ language and we will cover most of the relevant ones below. Courtesy by GeeksforGeeks

The C/C++ language has six primitive (meaning basic) data types used to store information, and they are:

  • char //Used to store only one alphanumeric character from ASCII encoding
  • String //Used to store multiple alphanumeric character from ASCII encoding, exists only in C++
  • int //Used to store only integers with a finite range 
  • long //Used to store only integers with a greater finite range than ints
  • float //Used to store decimal numbers, this can also include integers
  • void //A data type that is empty, useful with derived data types

The double back slashes "//" are known as comments, while "/* */" are multi-line comments and are notes for others to read and are ignored by the program.

Storing numbers:

The int, long and float are used to store numbers. But the method by which they are stored in the computer are different. Specifically, int and long can only store integers by representing decimal numbers as a bit sequence. See the diagram below and link to understand the calculations. Note that the range of integer depends on the processor used. The Arduino Uno with the ATMega328 chip stores an int as a length of 16 bits and a long as 32 bits.

Note that you raise two to the power of the position from the right, starting from zero and multiply it by 1 or 0 depending on the bit. Image was courteously shared by JavatPoint

The float stores decimal numbers. While you can store an integer such as: 9, it is stored differently on the computer than an int or long. See the diagram of a float data type below to understand its inner workings.

A floating point data type consists of:

  • A sign: that tells if the number is positive or negative
  • An exponent: the number that raises the power of 10
  • A fraction: where n is the position of the bit in the fractional component 12n.

The int or long does not have three parts to store numbers like the float and explaining calculations for the float is not needed to understand the Arduino completely. But more information can be found here.

Derived data types

Require a primitive data type to be used in your Arduino program and have specific syntaxes for their declaration. Let's start with the function and to declare one:

int my_function() {};

Where the first part of the code " int " is the primitive data type, and the second part " my_function " is the chosen name of the function and the " () {}; " refer to the contents of the function. We'll discuss their meaning more below.

Functions

Functions are code that modularise sections of code from another in C/C++ they must return a value at the end of their execution. They can be thought of as verbs because they do something, or they make an action occur. Opening the Arduino IDE greets you with two functions:

void setup() {

}

void loop() {
    
}

Again, using the function syntax we mentioned prior, " void " is an empty data type that will cause the function will return nothing at the end of executing its code. The use of " setup " is a name, and its purpose is to run the code inside the " {} " only once. More details be seen in the Arduino reference. Note the " () " are placeholders to input data (known as parameters or arguments) to the function and in this case is empty.

On the other hand, void loop() will repeatedly run its code between the " {} " , it does not take any parameters and returns nothing

Another function that you will commonly use encounter is "Serial.print("Hello World!") ", and serves to print the text " Hello world! " to the console of the Arduino IDE.

Note that the parameter is a text within a double quotations (" "), known as a string in programming (see the array section for the reason)

You'd be right to think that the function is odd because it has that Serial. at the beginning of the function because this function exists in a user defined data type known as a class. (Technically an object which we'll cover later)

Let's see "Serial.print("Hello World!") " in action with some code. (I suggest opening the IDE and typing in the code shown)

We first need to tell the Arduino and computer the rate at which they will communicate, by using the function Serial.begin(9600). 9600 is a standard rate the Arduino's and computers use to speak to each other and the function serves to only tell both PC and Arduino the rate of transmission. It also returns nothing.

We now have:

void setup() {
    Serial.begin(9600);
}

void loop() {
    
}

We can now add the Serial.print function as so:

void setup() {
    Serial.begin(9600);
    Serial.print("Hello world!");
}

void loop() {
    
}

Opening the serial monitor which appears as a magnifying glass, will reveal the text "Hello World!".

Using your own functions:

Notice that I've declared my own function myFunction below setup and loop. It's a good habit to declare any user made functions below these two because they are the most important functions in the Arduino.

Also keep in mind that myFunction will not be executed because I haven't specified it anywhere within void setup() or void loop(). Let's change that now:

void setup() {
    Serial.begin(9600);
    Serial.print("Hello world!");
    myFunction();
}

void loop() {

}

void myFunction() {
    
}

Now myFunction will be executed while doing nothing. If I add:

void setup() {
    Serial.begin(9600);
    Serial.print("Hello world!");
    myFunction();
}

void loop() {

}

void myFunction() {
    Serial.print("How are you!\n");
}

Which will print " How are you! " on a new line. Note that the \n is an ASCII character to put text on a new line.

void setup() {
    Serial.begin(9600);
    Serial.print("Hello world!");
    myFunction("Jay", 5);
}

void loop() {

}

void myFunction(String my_str, int my_num) {
    Serial.print("How are you!\n");
    Serial.print(my_str);
    Serial.print(my_num);
}

I've also added the parameters to the function String my_str and int my_num to print the name Jay and the number 5. You can add as many parameters to a functions as you wish.

Arrays

Arrays are a data structure that refers to a block of data with identitcal data types. They manifest themselves in RAM (Random Access Memory) where the data types are stored in sequential blocks.

The array refers to the entire block of data. The array elements are the individual blocks, and the array index always begins from 0 to allow programmers to access an array element.

You might wonder why we need arrays, and they're useful because they allow programmers to group related data instead of repeatedly declaring a new variable. Declaring an array:

void setup() {
    int my_array[5];
}

void loop() {

}

Declares the array my_array of five elements of the data type int and the contents of which are uninitialised (they have nothing inside them).

We can add contents to the array in two ways

Method 1: Adding each element on a new line separately:
void setup() {
    int my_array[5];
    my_array[0] = 1;
    my_array[1] = 2;
    my_array[2] = 4;
    my_array[3] = 8;
    my_array[4] = 16;
}

void loop() {

}

Which declares an array with five elements of the data type int. Note that if we tried to declare my_array[5] = 64; we would have an array that is "Out of bounds" because there will be six elements in the array. Remember we begin counting from zero in an array, otherwise it will result in an error.

Method 2: Declaring each element on the same line:
void setup() {
    int my_array[5] = {1, 2, 4, 8, 16};
}

void loop() {

}

And is functionally the same as method 1. Except the array elements are declared all in one line.

Method 1 and 2 visualised by showing the array contents. Note that we named our array my_array and not a.
